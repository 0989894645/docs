Optional minReadySeconds

minReadySeconds: undefined | number
  • Defined in types/input.ts:12472

Minimum number of seconds for which a newly created pod should be ready without any of its container crashing, for it to be considered available. Defaults to 0 (pod will be considered available as soon as it is ready)

Optional paused

paused: undefined | true | false
  • Defined in types/input.ts:12478

Indicates that the deployment is paused and will not be processed by the deployment controller.

Optional progressDeadlineSeconds

progressDeadlineSeconds: undefined | number
  • Defined in types/input.ts:12487

The maximum time in seconds for a deployment to make progress before it is considered to be failed. The deployment controller will continue to process failed deployments and a condition with a ProgressDeadlineExceeded reason will be surfaced in the deployment status. Note that progress will not be estimated during the time a deployment is paused. This is not set by default.

Optional replicas

replicas: undefined | number
  • Defined in types/input.ts:12493

Number of desired pods. This is a pointer to distinguish between explicit zero and not specified. Defaults to 1.

Optional revisionHistoryLimit

revisionHistoryLimit: undefined | number
  • Defined in types/input.ts:12499

The number of old ReplicaSets to retain to allow rollback. This is a pointer to distinguish between explicit zero and not specified.
